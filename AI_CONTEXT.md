# О ПРОЕКТЕ: Site-miniAPP (Mini-Apps Portal)

Этот файл служит основной базой знаний и точкой синхронизации для любого AI-ассистента, работающего с этим проектом. Перед внесением любых изменений обязательно изучите данный контекст.

## 🎯 Суть и назначение проекта
Проект представляет собой портал-каталог мини-приложений, игр и утилит, написанных в формате Single-Page HTML (где HTML, CSS и JS объединены в одном файле). 
Главная цель: все страницы, утилиты и игры должны **на 100% работать автономно в полном оффлайне** (например, в самолете, в бункере, без доступа к внешним серверам и CDN). 

### Структура проекта:
- `/apps/` — рабочие боевые файлы самих мини-приложений (например, `LetterSlot.html`, `WebAmp.html`).
- `/AppPages/` — страницы с описаниями приложений в каталоге, содержащие скриншоты и кнопку для оффлайн скачивания рабочего файла из `/apps/`.
- `/Utils/` — папки со вспомогательными PowerShell скриптами (Recoder, SORT_img, AudioExtr).
- `index.html` — главная страница каталога-портала.

## 🛠 Проделанная работа и текущее направление
В ходе предыдущих сессий мы проделали масштабную работу по стандартизации и стабилизации портала:

1. **Полная независимость от сети (Offline-First)**
   - Убраны все зависимости от внешних Google-шрифтов, осуществлен переход на системные шрифты (`system-ui, -apple-system, 'Segoe UI', Roboto, Helvetica, Arial, sans-serif`).
   - Удалены все внешние ссылки на скрипты/библиотеки. Вся логика, стили, звуки (через Web Audio API) генерируются и живут внутри самих файлов.

2. **Оффлайн скачивание через Base64 Data URL (КРИТИЧЕСКИ ВАЖНО)**
   - На страницах-описаниях в `AppPages/*.html` есть кнопки СКАЧАТЬ ДЛЯ ОФФЛАЙН (`btn-offline-download`).
   - Чтобы скачивание файлов `apps/*.html` работало без интернета и локального веб-сервера (когда проект открыт прямо через `file:///C:/...` в браузере Chrome/Edge), мы используем Base64. Весь код целевого приложения (`apps/*.html`) кодируется в Base64 и вставляется в `href` кнопки на странице описания.

3. **Локализация и стабильная верстка**
   - Файлы переименованы строго на латиницу без пробелов (например, `Буквенный слот.html` -> `LetterSlot.html`), чтобы избежать проблем с кодировкой путей Windows.
   - В игру `LetterSlot.html` добавлена мультиязычность (RU, EN, TR). Верстка страниц сделана жесткой с помощью минимальной высоты блоков (min-height), чтобы смена языков не вызывала сдвигов интерфейса.

4. **Эстетика и Скриншоты**
   - Визуальный стиль проекта — теплые аккуратные градиенты, Glassmorphism, мягкие закругления кнопок (`1.25rem`). 
   - Скриншоты в папке `AppPages/screenshots/` пересняты и строго кадрированы по контуру самого приложения (без черных внешних полей).

5. **Хостинг на GitHub Pages (Важные требования)**
   - Сайт будет размещаться на GitHub Pages, поэтому архитектура должна строго соответствовать требованиям статического хостинга.
   - **Регистр символов (Case-sensitivity):** GitHub Pages работает на Linux-серверах, поэтому пути и названия файлов строго чувствительны к регистру! Файл `LetterSlot.html` и `letterslot.html` — это два разных файла. Ссылки в коде должны на 100% совпадать с реальным именем файла, иначе на хостинге будут ошибки 404 (даже если на Windows всё работало).
   - **Только статика:** Сервис поддерживает только отдачу статических файлов (HTML, CSS, JS). Запрещено использовать backend-технологии (PHP, Node.js базы данных).
   - **Игнорирование файлов:** GH Pages по умолчанию (через Jekyll) игнорирует файлы и папки, начинающиеся с нижнего подчеркивания (`_`). Придерживайся стандартных имен без спецсимволов.

## 🛑 ЖЕСТКИЕ ПРАВИЛА ДЛЯ AI АССИСТЕНТОВ (ВНИМАНИЕ!)

1. **СОХРАНЯЙ И КОММИТЬ ВЕРСИИ:** При внесении изменений, особенно если это крупные правки, затрагивающие несколько файлов, или сложный рефакторинг логики — **ВСЕГДА СОХРАНЯЙ ВЕРСИИ И ДЕЛАЙ КОММИТЫ (COMMIT) С ОПИСАНИЕМ**. Никаких слепых перезаписей рабочих компонентов без возможности отката! Сохраняй историю, чтобы пользователь мог откатить проект при сбое.
2. **ОБНОВЛЯЙ BASE64 ПРИ ПРАВКАХ:** Если ты изменил код любого приложения внутри папки `/apps/` (например, `LetterSlot.html`), ты **ОБЯЗАН** переупаковать этот измененный файл в Base64 Data URL и обновить атрибут `href` кнопки скачивания на соответствующей странице описания в папке `/AppPages/`. Иначе пользователь скачает устаревшую версию.
3. **НЕ ДОБАВЛЯЙ ВНЕШНИЕ CDN:** Никакого подключения сторонних библиотек, шрифтов, jQuery, Bootstrap, картинок с внешних ссылок и т.д. Проект обязан жить без интернета.
4. **СЛЕДИ ЗА КОДИРОВКОЙ И НАЗВАНИЯМИ:** Все новые файлы должны называться латинскими буквами без пробелов. Кодировка всех HTML файлов строго UTF-8 (без BOM).
5. **ТОЧЕЧНЫЕ ПРАВКИ:** Редактируй только те части файлов, которые просит пользователь (Surgical Changes). Не "улучшай" не связанный с задачей код.
